# HireHi: Анализ IT-вакансий платформы HireHi
## Что сделали в этой части ГП: 
Собрали данные с платформы HireHi c помощью requests и BeautifulSoup. Собрали датасет с информацией о вакансиях, выполнили базовый EDA и сделали несколько визуализаций. 

### 0. Сбор данных. Признаки
* **`id`** *(Тип: int64)* - Уникальный идентификатор вакансии.
* **`title`** *(Тип: object)* - Исходное название вакансии (например, "Backend разработчик (Python)"), признак с множеством уникальных значений.
* **`category`** *(Тип: object)* - Направление вакансии. Всего собрано 7 категорий: **development, analytics, management, qa, design, devops marketing** 
* **`company`** *(Тип: object)* - Название компании-работодателя. Признак с множеством уникальных значений.
* **`format`** *(Тип: object)* - Формат работы. Содержит множество уникальных значений **(Например, 'офис Астана','гибрид Питер')**.
* **`level`** *(Тип: object)* - Обозначает требуемый грейд специалиста. Всего 6 грейдов: **intern, junior, middle, lead, senior, head**.
* **`salary`** *(Тип: object)* - Исходный тип приводится к **("float64")** Предлагаемая заработная плата в рублях(диапазон, либо конкретное значение)
* **`link`** *(Тип: object)* - Прямая ссылка на страницу вакансии.
* **`skills`** *(Тип: object)* - Требуемый технологический стек (например, **"Python, SQL, Docker, FastAPI"**)

В процессе обработки исходный "format" был удален и были созданы два новых признака:
* **`format`** *(Тип: object)* - Формат работы. Содержит три основные категории: **удалённо**, **офис**, **гибрид**.
* **`city/country`** *(Тип: object)* - Уточненная локация (город или страна, включая зарубежные локации, например, "США").

### 3. Пропущенные значения и текстовые заполнения
* В столбце **`salary`** изначально присутствовало 46 стандартных пропусков (`NaN`). 
* Кроме `NaN` в признаке `salary` использовались текстовые заполнения для обозначения отсутствия данных о доходе, например, `"зп не указана"` и `"не оплачивается"`. С их учетом общее количество пропусков составило 266.

### 4. Особенности признака `salary`
* Изначально зарплаты представлены текстом с указанием вилок (например, `от 180 до 220 ₽`, `200 000 - 285 000`), приблизительных значений (`~ 372 000 ₽`) или только нижних границ (`от 240 000 ₽`).
* Все зарплаты указаны в рублях, вакансии с оплатой в долларах и других валютах отсутствуют.
* В ходе предобработки из строк были удалены все нечисловые символы. Для вилок вычислено среднее арифметическое значение. Зарплаты, изначально указанные в тысячах (значения <= 1000, например `180` вместо `180000`), умножены на 1000 для приведения к единому масштабу.
* Итоговые пропуски в зарплатах (266 записей) заполнены медианными значениями, рассчитанными индивидуально для каждой группы на основе комбинации категории (`category`) и грейда (`level`).

### 5. Обработан и изучен признак `skills`
* Изначально данные находятся в строковом формате (например, `["Python", "SQL", "Docker", "FastAPI"]`) - проведена обработка к формату списка.
* Проведена оценка пропусков, наличия дубликатов - найдено `55` пропусков, в последствии удалены из-за незначительной доли.
* Найдены самые распространынные навыки, а также навыки приносящие наибольший доход.
